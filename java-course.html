1. Pendahuluan Java

Definisi Java

Java adalah bahasa pemrograman berorientasi objek (OOP) yang dikembangkan oleh Sun Microsystems (sekarang dimiliki oleh Oracle) pada tahun 1995. Java dirancang dengan filosofi "Write Once, Run Anywhere" (WORA), yang memungkinkan kode Java dijalankan di berbagai platform tanpa perlu dikompilasi ulang.

Keunggulan Java

  • Platform Independen: Kode Java dapat dijalankan di berbagai sistem operasi yang memiliki Java Virtual Machine (JVM).
  • Pustaka yang Luas: Java memiliki koleksi pustaka standar yang kaya, memudahkan pengembangan berbagai jenis aplikasi.
  • Penggunaan Luas: Java digunakan dalam berbagai bidang, mulai dari aplikasi desktop, aplikasi web, aplikasi mobile (Android), hingga sistem embedded.
Catatan

Java adalah salah satu bahasa pemrograman paling populer di dunia, dengan jutaan pengembang aktif dan digunakan oleh perusahaan-perusahaan besar seperti Google, Amazon, dan Netflix.

2. Struktur Program Java

Class: Blueprint untuk objek

Class adalah blueprint atau template yang mendefinisikan atribut dan perilaku objek. Setiap program Java minimal memiliki satu class. Class menjadi dasar dari pemrograman berorientasi objek di Java.

Method: Blok kode untuk tugas spesifik

Method adalah blok kode yang melakukan tugas tertentu dan dapat dipanggil untuk dieksekusi. Method dapat menerima parameter dan mengembalikan nilai. Method memungkinkan kita untuk mengorganisir kode menjadi unit-unit fungsional yang dapat digunakan kembali.

Main Method: Titik awal eksekusi program

Method main adalah titik awal eksekusi program Java. Setiap aplikasi Java yang dapat dijalankan harus memiliki method main. JVM akan mencari method main sebagai titik masuk untuk menjalankan program.

BasicStructure.java 
// Deklarasi class
public class BasicStructure {
    
    // Variabel instance
    private String message;
    
    // Constructor
    public BasicStructure(String message) {
        this.message = message;
    }
    
    // Method
    public void displayMessage() {
        System.out.println(message);
    }
    
    // Main method - titik awal eksekusi
    public static void main(String[] args) {
        // Membuat objek dari class BasicStructure
        BasicStructure obj = new BasicStructure("Halo, Dunia Java!");
        
        // Memanggil method
        obj.displayMessage();
    }
}

3. Mengupas Tuntas Main Method

Main method memiliki sintaks khusus yang harus diikuti agar program Java dapat dijalankan:

MainMethod.java 
public static void main(String[] args) {
    // Kode program
}

public: Modifier akses

Keyword public menunjukkan bahwa method main dapat diakses dari mana saja, termasuk oleh JVM saat menjalankan program. JVM harus dapat mengakses method main, sehingga modifier akses harus public.

static: Digunakan tanpa instansiasi objek

Keyword static memungkinkan method main dipanggil tanpa perlu membuat instance dari class. JVM dapat langsung memanggil method main tanpa membuat objek terlebih dahulu.

void: Tidak mengembalikan nilai

Keyword void menunjukkan bahwa method main tidak mengembalikan nilai apapun. Method main dijalankan oleh JVM dan tidak perlu mengembalikan nilai.

String[] args: Parameter command line

Parameter String[] args adalah array yang berisi argumen command line yang diberikan saat menjalankan program. Ini memungkinkan program menerima input dari pengguna saat dijalankan dari command line.

CommandLineArgs.java 
public class CommandLineArgs {
    public static void main(String[] args) {
        System.out.println("Jumlah argumen: " + args.length);
        
        for (int i = 0; i < args.length; i++) {
            System.out.println("Argumen " + i + ": " + args[i]);
        }
    }
}

4. Public vs Private Modifier

Java memiliki beberapa modifier akses yang menentukan visibilitas dan aksesibilitas class, variabel, dan method. Dua modifier akses yang paling umum digunakan adalah public dan private.

Public: Dapat diakses dari mana saja

Modifier public memungkinkan akses dari class manapun, baik dalam package yang sama maupun berbeda. Ini adalah modifier akses yang paling terbuka.

  • Class, method, atau variabel dengan modifier public dapat diakses dari mana saja
  • Biasanya digunakan untuk API yang ingin diekspos ke pengguna lain
  • Main method harus public agar dapat diakses oleh JVM

Private: Hanya dapat diakses dalam kelas

Modifier private membatasi akses hanya dari dalam class yang sama. Ini adalah bentuk enkapsulasi yang paling ketat.

  • Method atau variabel dengan modifier private hanya dapat diakses dari dalam class yang sama
  • Biasanya digunakan untuk menyembunyikan implementasi internal dan melindungi data
  • Merupakan praktik yang baik untuk membuat variabel instance sebagai private dan menyediakan method getter dan setter untuk mengaksesnya
AccessModifiers.java 
public class AccessModifiers {
    // Dapat diakses dari mana saja
    public String publicVar = "Ini variabel public";
    
    // Hanya dapat diakses dalam class ini
    private String privateVar = "Ini variabel private";
    
    // Method public
    public void publicMethod() {
        System.out.println("Ini method public");
        // Method public dapat mengakses variabel private
        System.out.println(privateVar);
    }
    
    // Method private
    private void privateMethod() {
        System.out.println("Ini method private");
    }
    
    // Method untuk mendemonstrasikan akses
    public void demonstrateAccess() {
        // Dapat memanggil method private dari dalam class yang sama
        privateMethod();
    }
    
    public static void main(String[] args) {
        AccessModifiers obj = new AccessModifiers();
        
        // Akses ke variabel dan method public
        System.out.println(obj.publicVar);
        obj.publicMethod();
        
        // Akses ke method yang memanggil method private
        obj.demonstrateAccess();
        
        // Kode berikut akan menyebabkan error kompilasi:
        // System.out.println(obj.privateVar);
        // obj.privateMethod();
    }
}
Catatan

Selain public dan private, Java juga memiliki modifier akses lain seperti protected (dapat diakses dalam package yang sama dan subclass) dan default/package-private (dapat diakses dalam package yang sama).

5. Static Keyword

Keyword static digunakan untuk membuat variabel dan method yang bersifat global dalam kelas, artinya mereka milik class, bukan milik instance (objek) dari class tersebut.

Kapan Harus Menggunakan Static

  • Variabel yang Sama untuk Semua Objek: Gunakan static untuk variabel yang nilainya harus sama untuk semua objek dari class tersebut.
  • Konstanta: Variabel static final sering digunakan untuk mendefinisikan konstanta.
  • Utility Methods: Method yang tidak bergantung pada state objek dan hanya melakukan operasi pada parameter yang diberikan.
  • Factory Methods: Method static yang mengembalikan instance dari class tersebut.
  • Main Method: Method main harus static agar dapat dijalankan oleh JVM tanpa membuat instance dari class.
StaticExample.java 
public class StaticExample {
    // Static variable (konstanta)
    public static final double PI = 3.14159;
    
    // Static counter
    private static int counter = 0;
    
    // Instance variable
    private String name;
    
    // Constructor
    public StaticExample(String name) {
        this.name = name;
        // Increment counter setiap kali objek dibuat
        counter++;
    }
    
    // Static method
    public static int getCounter() {
        return counter;
    }
    
    // Static method untuk kalkulasi matematika
    public static double calculateCircleArea(double radius) {
        return PI * radius * radius;
    }
    
    // Non-static method
    public void displayInfo() {
        System.out.println("Nama: " + name);
        System.out.println("Jumlah objek: " + counter);
    }
    
    public static void main(String[] args) {
        // Mengakses static method dan variable tanpa membuat objek
        System.out.println("PI: " + StaticExample.PI);
        System.out.println("Luas lingkaran: " + StaticExample.calculateCircleArea(5));
        
        // Membuat beberapa objek
        StaticExample obj1 = new StaticExample("Objek 1");
        StaticExample obj2 = new StaticExample("Objek 2");
        StaticExample obj3 = new StaticExample("Objek 3");
        
        // Menampilkan informasi
        obj1.displayInfo();
        
        // Mengakses static counter
        System.out.println("Total objek: " + StaticExample.getCounter());
    }
}
Penting

Method static tidak dapat mengakses variabel instance (non-static) atau memanggil method non-static secara langsung, karena method static tidak memiliki referensi ke instance tertentu dari class.

6. Void vs Non-Void Methods

Void: Tidak mengembalikan nilai

Method dengan tipe pengembalian void tidak mengembalikan nilai apapun. Method ini biasanya digunakan untuk melakukan tindakan atau operasi tanpa perlu mengembalikan hasil.

  • Digunakan ketika method hanya perlu melakukan tugas tanpa mengembalikan nilai
  • Contoh: method untuk mencetak, mengubah state objek, atau melakukan operasi I/O
  • Tidak menggunakan statement return dengan nilai (bisa menggunakan return; untuk keluar dari method lebih awal)

Non-Void: Mengembalikan nilai menggunakan return

Method non-void mengembalikan nilai dengan tipe data tertentu. Method ini harus menggunakan statement return untuk mengembalikan nilai yang sesuai dengan tipe pengembalian yang dideklarasikan.

  • Digunakan ketika method perlu menghitung atau menghasilkan nilai
  • Tipe pengembalian bisa berupa tipe primitif (int, double, boolean, dll.) atau tipe referensi (String, array, objek, dll.)
  • Harus memiliki statement return yang mengembalikan nilai dengan tipe yang sesuai
MethodTypes.java 
public class MethodTypes {
    // Void method - tidak mengembalikan nilai
    public void greet(String name) {
        System.out.println("Halo, " + name + "!");
        // Tidak ada return value
    }
    
    // Non-void method - mengembalikan nilai int
    public int add(int a, int b) {
        return a + b; // Mengembalikan hasil penjumlahan
    }
    
    // Non-void method - mengembalikan nilai boolean
    public boolean isEven(int number) {
        return number % 2 == 0; // Mengembalikan true jika genap, false jika ganjil
    }
    
    // Non-void method - mengembalikan objek String
    public String formatName(String firstName, String lastName) {
        return lastName + ", " + firstName;
    }
    
    public static void main(String[] args) {
        MethodTypes demo = new MethodTypes();
        
        // Memanggil void method
        demo.greet("Eben"); // Output: Halo, Eben!
        
        // Memanggil non-void methods dan menggunakan nilai yang dikembalikan
        int sum = demo.add(5, 3);
        System.out.println("5 + 3 = " + sum); // Output: 5 + 3 = 8
        
        boolean isEight = demo.isEven(8);
        System.out.println("Apakah 8 genap? " + isEight); // Output: Apakah 8 genap? true
        
        String formattedName = demo.formatName("John", "Doe");
        System.out.println("Nama: " + formattedName); // Output: Nama: Doe, John
    }
}

7. String[] args: Opsional atau Wajib?

Parameter String[] args dalam method main digunakan untuk menerima argumen command line saat program dijalankan. Pertanyaannya, apakah parameter ini opsional atau wajib?

Wajib, Tapi Namanya Bisa Diubah

Parameter array String dalam method main adalah wajib, tetapi nama parameternya (args) bisa diubah menjadi nama lain seperti arguments, params, dll. Yang penting adalah tipe datanya harus String[].

MainMethodVariations.java 
public class MainMethodVariations {
    // Nama parameter bisa diubah
    public static void main(String[] arguments) {
        System.out.println("Jumlah argumen: " + arguments.length);
    }
    
    // Ini juga valid
    // public static void main(String[] params) {
    //     System.out.println("Jumlah argumen: " + params.length);
    // }
    
    // Ini juga valid - menggunakan varargs (variable arguments)
    // public static void main(String... args) {
    //     System.out.println("Jumlah argumen: " + args.length);
    // }
}

Tidak Bisa Dihapus

Jika parameter String[] args dihapus, maka method main tidak akan dikenali oleh JVM sebagai entry point program, dan akan menyebabkan error saat kompilasi atau runtime.

InvalidMain.java 
public class InvalidMain {
    // Ini TIDAK valid sebagai entry point program
    public static void main() {
        System.out.println("Ini tidak akan dijalankan sebagai entry point");
    }
    
    // Method main yang benar harus memiliki parameter String[]
    public static void main(String[] args) {
        System.out.println("Ini adalah entry point yang valid");
    }
}

Cara Menggunakan Command Line Arguments

Untuk menjalankan program Java dengan argumen command line, gunakan format berikut:

Command Line 
java ProgramName arg1 arg2 arg3

Contoh program yang menggunakan argumen command line:

Calculator.java 
public class Calculator {
    public static void main(String[] args) {
        if (args.length != 3) {
            System.out.println("Penggunaan: java Calculator angka1 operator angka2");
            System.out.println("Operator yang didukung: +, -, *, /");
            return;
        }
        
        try {
            double num1 = Double.parseDouble(args[0]);
            String operator = args[1];
            double num2 = Double.parseDouble(args[2]);
            double result = 0;
            
            switch (operator) {
                case "+":
                    result = num1 + num2;
                    break;
                case "-":
                    result = num1 - num2;
                    break;
                case "*":
                    result = num1 * num2;
                    break;
                case "/":
                    if (num2 == 0) {
                        System.out.println("Error: Pembagian dengan nol");
                        return;
                    }
                    result = num1 / num2;
                    break;
                default:
                    System.out.println("Operator tidak valid: " + operator);
                    return;
            }
            
            System.out.println(num1 + " " + operator + " " + num2 + " = " + result);
        } catch (NumberFormatException e) {
            System.out.println("Error: Argumen harus berupa angka");
        }
    }
}
Catatan

Meskipun parameter String[] args wajib ada, program Anda tidak harus menggunakannya. Jika program Anda tidak memerlukan argumen command line, Anda bisa mengabaikan parameter tersebut dalam implementasi method main.

8. Variabel dan Tipe Data

Java adalah bahasa yang strongly typed, artinya setiap variabel harus dideklarasikan dengan tipe data tertentu. Java memiliki dua kategori utama tipe data: primitif dan non-primitif (referensi).

Tipe Data Primitif

Tipe data primitif adalah tipe data dasar yang disediakan oleh Java. Ada 8 tipe data primitif:

Tipe Data Ukuran Rentang Nilai Contoh
byte 8-bit -128 sampai 127 byte b = 100;
short 16-bit -32,768 sampai 32,767 short s = 1000;
int 32-bit -2^31 sampai 2^31-1 int i = 100000;
long 64-bit -2^63 sampai 2^63-1 long l = 100000L;
float 32-bit ±3.4e−038 sampai ±3.4e+038 float f = 3.14f;
double 64-bit ±1.7e−308 sampai ±1.7e+308 double d = 3.14159;
char 16-bit 0 sampai 65,535 (karakter Unicode) char c = 'A';
boolean 1-bit true atau false boolean b = true;

Tipe Data Non-Primitif (Referensi)

Tipe data non-primitif adalah tipe data yang merujuk ke objek. Beberapa tipe data non-primitif yang umum digunakan:

  • String: Untuk merepresentasikan teks
  • Array: Untuk menyimpan kumpulan nilai dengan tipe yang sama
  • Class: Tipe data yang didefinisikan pengguna
  • Interface: Blueprint untuk class
  • Wrapper Classes: Kelas yang membungkus tipe primitif (Integer, Double, Boolean, dll.)
DataTypes.java 
public class DataTypes {
    public static void main(String[] args) {
        // Tipe data primitif
        byte byteVar = 100;
        short shortVar = 1000;
        int intVar = 100000;
        long longVar = 100000000L;  // Perhatikan suffix 'L'
        
        float floatVar = 3.14f;     // Perhatikan suffix 'f'
        double doubleVar = 3.14159;
        
        char charVar = 'A';
        boolean boolVar = true;
        
        // Tipe data non-primitif
        String stringVar = "Hello, Java!";
        
        // Array
        int[] intArray = {1, 2, 3, 4, 5};
        
        // Wrapper classes
        Integer integerObj = 100;
        Double doubleObj = 3.14159;
        
        // Menampilkan nilai variabel
        System.out.println("byte: " + byteVar);
        System.out.println("short: " + shortVar);
        System.out.println("int: " + intVar);
        System.out.println("long: " + longVar);
        System.out.println("float: " + floatVar);
        System.out.println("double: " + doubleVar);
        System.out.println("char: " + charVar);
        System.out.println("boolean: " + boolVar);
        System.out.println("String: " + stringVar);
        System.out.println("Array: " + intArray[0] + ", " + intArray[1] + ", ...");
        System.out.println("Integer object: " + integerObj);
        System.out.println("Double object: " + doubleObj);
    }
}
Perbedaan Tipe Primitif dan Non-Primitif

Tipe primitif menyimpan nilai secara langsung, sedangkan tipe non-primitif menyimpan referensi ke objek. Tipe primitif memiliki nilai default (0, 0.0, false, '\u0000'), sedangkan tipe non-primitif memiliki nilai default null.

9. Float vs Double

Float dan double adalah dua tipe data primitif yang digunakan untuk menyimpan angka desimal (floating-point) di Java. Keduanya memiliki perbedaan dalam hal ukuran memori dan presisi.

Float: 32-bit, presisi 6-7 digit

  • Menggunakan 32 bit (4 byte) memori
  • Memiliki presisi sekitar 6-7 digit desimal
  • Rentang nilai: ±3.4e−038 sampai ±3.4e+038
  • Harus menggunakan suffix 'f' atau 'F' (contoh: 3.14f)
  • Lebih hemat memori, tetapi kurang presisi

Double: 64-bit, presisi 15-16 digit

  • Menggunakan 64 bit (8 byte) memori
  • Memiliki presisi sekitar 15-16 digit desimal
  • Rentang nilai: ±1.7e−308 sampai ±1.7e+308
  • Tidak memerlukan suffix (opsional: 'd' atau 'D')
  • Lebih presisi, tetapi menggunakan lebih banyak memori
FloatVsDouble.java 
public class FloatVsDouble {
    public static void main(String[] args) {
        // Deklarasi dan inisialisasi
        float floatVar = 3.14159265359f;  // Perhatikan suffix 'f'
        double doubleVar = 3.14159265359;  // Tanpa suffix
        
        // Menampilkan nilai
        System.out.println("Float: " + floatVar);  // Output: 3.1415927 (presisi terbatas)
        System.out.println("Double: " + doubleVar);  // Output: 3.14159265359
        
        // Demonstrasi presisi
        float floatSum = 0.0f;
        double doubleSum = 0.0;
        
        for (int i = 0; i < 1000000; i++) {
            floatSum += 0.0001f;
            doubleSum += 0.0001;
        }
        
        System.out.println("Hasil penjumlahan 0.0001 sebanyak 1.000.000 kali:");
        System.out.println("Float: " + floatSum);  // Mungkin tidak tepat 100
        System.out.println("Double: " + doubleSum);  // Lebih mendekati 100
        
        // Perbandingan ukuran
        System.out.println("Ukuran float: " + Float.SIZE / 8 + " bytes");
        System.out.println("Ukuran double: " + Double.SIZE / 8 + " bytes");
    }
}

Kapan Menggunakan Float vs Double

Pilihan antara float dan double tergantung pada kebutuhan aplikasi:

  • Gunakan float ketika:
    • Memori adalah pertimbangan utama
    • Presisi tinggi tidak diperlukan
    • Bekerja dengan array besar atau struktur data
    • Aplikasi grafis atau game yang memerlukan kecepatan
  • Gunakan double ketika:
    • Presisi tinggi diperlukan
    • Melakukan perhitungan ilmiah atau finansial
    • Bekerja dengan angka yang sangat besar atau sangat kecil
    • Memori bukan masalah utama
Catatan Penting

Secara default, Java menggunakan double untuk literal floating-point. Jika Anda ingin menggunakan float, Anda harus menambahkan suffix 'f' atau 'F'. Contoh: float f = 3.14f;

10. Operator dalam Java

Operator adalah simbol khusus yang digunakan untuk melakukan operasi pada variabel dan nilai. Java memiliki beberapa jenis operator yang dapat digunakan untuk berbagai keperluan.

Operator Aritmatika

Digunakan untuk melakukan operasi matematika dasar.

Operator Deskripsi Contoh
+ Penjumlahan int c = a + b;
- Pengurangan int c = a - b;
* Perkalian int c = a * b;
/ Pembagian int c = a / b;
% Modulus (sisa pembagian) int c = a % b;
++ Increment a++; atau ++a;
-- Decrement a--; atau --a;

Operator Perbandingan

Digunakan untuk membandingkan dua nilai atau variabel. Hasil dari operator perbandingan adalah nilai boolean (true atau false).

Operator Deskripsi Contoh
== Sama dengan if (a == b) {...}
!= Tidak sama dengan if (a != b) {...}
> Lebih besar dari if (a > b) {...}
< Lebih kecil dari if (a < b) {...}
>= Lebih besar dari atau sama dengan if (a >= b) {...}
<= Lebih kecil dari atau sama dengan if (a <= b) {...}

Operator Logika

Digunakan untuk melakukan operasi logika pada nilai boolean.

Operator Deskripsi Contoh
&& AND logika (kedua kondisi harus true) if (a > 5 && b < 10) {...}
|| OR logika (salah satu kondisi harus true) if (a > 5 || b < 10) {...}
! NOT logika (membalikkan nilai boolean) if (!isValid) {...}
OperatorDemo.java 
public class OperatorDemo {
    public static void main(String[] args) {
        // Operator Aritmatika
        int a = 10;
        int b = 3;
        
        System.out.println("a + b = " + (a + b));  // 13
        System.out.println("a - b = " + (a - b));  // 7
        System.out.println("a * b = " + (a * b));  // 30
        System.out.println("a / b = " + (a / b));  // 3 (pembagian integer)
        System.out.println("a % b = " + (a % b));  // 1 (sisa pembagian)
        
        // Increment dan Decrement
        int c = 5;
        System.out.println("c++ = " + (c++));  // 5 (post-increment: nilai c digunakan dulu, baru ditambah)
        System.out.println("c sekarang = " + c);  // 6
        
        int d = 5;
        System.out.println("++d = " + (++d));  // 6 (pre-increment: nilai d ditambah dulu, baru digunakan)
        
        // Operator Perbandingan
        System.out.println("a == b: " + (a == b));  // false
        System.out.println("a != b: " + (a != b));  // true
        System.out.println("a > b: " + (a > b));    // true
        System.out.println("a < b: " + (a < b));    // false
        System.out.println("a >= b: " + (a >= b));  // true
        System.out.println("a <= b: " + (a <= b));  // false
        
        // Operator Logika
        boolean x = true;
        boolean y = false;
        
        System.out.println("x && y: " + (x && y));  // false
        System.out.println("x || y: " + (x || y));  // true
        System.out.println("!x: " + (!x));          // false
        
        // Operator Bitwise (untuk operasi bit)
        System.out.println("a & b: " + (a & b));    // 2 (bitwise AND)
        System.out.println("a | b: " + (a | b));    // 11 (bitwise OR)
        System.out.println("a ^ b: " + (a ^ b));    // 9 (bitwise XOR)
        System.out.println("~a: " + (~a));          // -11 (bitwise NOT)
        System.out.println("a << 2: " + (a << 2));  // 40 (left shift)
        System.out.println("a >> 2: " + (a >> 2));  // 2 (right shift)
    }
}

11. Looping (Perulangan)

Perulangan (looping) digunakan untuk mengeksekusi blok kode berulang kali selama kondisi tertentu terpenuhi. Java menyediakan beberapa jenis perulangan untuk berbagai kebutuhan.

For Loop: Inisialisasi, kondisi, increment/decrement

For loop digunakan ketika kita tahu berapa kali kita ingin mengeksekusi blok kode. For loop memiliki tiga bagian: inisialisasi, kondisi, dan increment/decrement.

ForLoopExample.java 
// Sintaks dasar for loop
for (inisialisasi; kondisi; increment/decrement) {
    // blok kode yang akan dieksekusi
}

// Contoh for loop
for (int i = 0; i < 5; i++) {
    System.out.println("Iterasi ke-" + i);
}

// For loop dengan multiple variabel
for (int i = 0, j = 10; i < j; i++, j--) {
    System.out.println("i = " + i + ", j = " + j);
}

While Loop: Mengeksekusi selama kondisi bernilai benar

While loop digunakan ketika kita ingin mengeksekusi blok kode selama kondisi tertentu bernilai true. Kondisi dievaluasi sebelum blok kode dieksekusi.

WhileLoopExample.java 
// Sintaks dasar while loop
while (kondisi) {
    // blok kode yang akan dieksekusi
}

// Contoh while loop
int i = 0;
while (i < 5) {
    System.out.println("Iterasi ke-" + i);
    i++;
}

// While loop dengan kondisi kompleks
int num = 1;
while (num <= 100 && num % 17 != 0) {
    System.out.println("Angka: " + num);
    num++;
}

Do-While Loop: Minimal dijalankan sekali sebelum memeriksa kondisi

Do-while loop mirip dengan while loop, tetapi kondisi dievaluasi setelah blok kode dieksekusi. Ini berarti blok kode akan dieksekusi minimal satu kali, bahkan jika kondisi awalnya false.

DoWhileLoopExample.java 
// Sintaks dasar do-while loop
do {
    // blok kode yang akan dieksekusi
} while (kondisi);

// Contoh do-while loop
int i = 0;
do {
    System.out.println("Iterasi ke-" + i);
    i++;
} while (i < 5);

// Do-while loop yang dieksekusi minimal sekali
int j = 10;
do {
    System.out.println("Nilai j: " + j);
    j++;
} while (j < 5);  // Kondisi false, tetapi blok kode tetap dieksekusi sekali

Enhanced For Loop (For-Each)

Enhanced for loop (juga disebut for-each loop) digunakan untuk mengiterasi elemen dalam array atau collection. Ini lebih sederhana dan lebih mudah dibaca daripada for loop tradisional untuk kasus ini.

EnhancedForLoopExample.java 
// Sintaks dasar enhanced for loop
for (tipeData variabel : array/collection) {
    // blok kode yang akan dieksekusi
}

// Contoh enhanced for loop dengan array
int[] numbers = {1, 2, 3, 4, 5};
for (int num : numbers) {
    System.out.println("Angka: " + num);
}

// Contoh enhanced for loop dengan collection
ArrayList names = new ArrayList<>();
names.add("John");
names.add("Jane");
names.add("Bob");

for (String name : names) {
    System.out.println("Nama: " + name);
}

Kontrol Perulangan: break dan continue

Java menyediakan dua statement untuk mengontrol perulangan: break dan continue.

  • break: Menghentikan perulangan dan keluar dari blok perulangan
  • continue: Melewati iterasi saat ini dan melanjutkan ke iterasi berikutnya
LoopControlExample.java 
// Contoh penggunaan break
for (int i = 0; i < 10; i++) {
    if (i == 5) {
        break;  // Keluar dari loop saat i = 5
    }
    System.out.println("Break example: " + i);
}

// Contoh penggunaan continue
for (int i = 0; i < 10; i++) {
    if (i % 2 == 0) {
        continue;  // Lewati iterasi jika i genap
    }
    System.out.println("Continue example: " + i);  // Hanya mencetak angka ganjil
}
Tips Memilih Jenis Perulangan
  • Gunakan for loop ketika Anda tahu berapa kali perulangan akan dijalankan
  • Gunakan while loop ketika Anda tidak tahu berapa kali perulangan akan dijalankan, tetapi tahu kondisi berhentinya
  • Gunakan do-while loop ketika Anda ingin blok kode dieksekusi minimal satu kali
  • Gunakan enhanced for loop ketika Anda ingin mengiterasi semua elemen dalam array atau collection

12. Memahami Array

Array adalah struktur data yang digunakan untuk menyimpan kumpulan nilai dengan tipe data yang sama. Array memiliki ukuran tetap yang ditentukan saat pembuatan.

Deklarasi dan Inisialisasi Array

Ada beberapa cara untuk mendeklarasikan dan menginisialisasi array di Java:

ArrayDeclaration.java 
// Deklarasi array
int[] numbers;  // Cara yang direkomendasikan
int numbers[];  // Cara alternatif (gaya C/C++)

// Inisialisasi array dengan ukuran tertentu

Akses Elemen dengan Indeks

Elemen array diakses menggunakan indeks, yang dimulai dari 0 untuk elemen pertama.

System.out.println("Elemen di baris 1, kolom 2: " + matrix[1][2]);  // 6

matrix[0][1] = 10;

Looping Array dengan for dan for-each

Ada beberapa cara untuk mengiterasi elemen dalam array:

ArrayLooping.java 
int[] numbers = {10, 20, 30, 40, 50};

// Menggunakan for loop tradisional
System.out.println("Menggunakan for loop:");
for (int i = 0; i < numbers.length; i++) {
    System.out.println("Elemen ke-" + i + ": " + numbers[i]);
}

// Menggunakan enhanced for loop (for-each)
System.out.println("\nMenggunakan enhanced for loop:");
for (int num : numbers) {
    System.out.println("Elemen: " + num);
}

// Looping array 2D dengan for loop tradisional
int[][] matrix = {
    {1, 2, 3},
    {4, 5, 6},
    {7, 8, 9}
};

System.out.println("\nMenggunakan for loop untuk array 2D:");
for (int i = 0; i < matrix.length; i++) {
    for (int j = 0; j < matrix[i].length; j++) {
        System.out.print(matrix[i][j] + " ");
    }
    System.out.println();  // Pindah ke baris baru setelah setiap baris matrix
}

// Looping array 2D dengan enhanced for loop
System.out.println("\nMenggunakan enhanced for loop untuk array 2D:");
for (int[] row : matrix) {
    for (int element : row) {
        System.out.print(element + " ");
    }
    System.out.println();
}

Utilitas Array

Java menyediakan kelas java.util.Arrays yang berisi berbagai method utilitas untuk bekerja dengan array.

ArrayUtils.java 
import java.util.Arrays;

public class ArrayUtils {
    public static void main(String[] args) {
        int[] numbers = {5, 2, 9, 1, 7, 3};
        
        // Mengurutkan array
        Arrays.sort(numbers);
        System.out.println("Array setelah diurutkan: " + Arrays.toString(numbers));
        
        // Mencari elemen dalam array yang sudah diurutkan
        int index = Arrays.binarySearch(numbers, 7);
        System.out.println("Indeks elemen 7: " + index);
        
        // Membandingkan dua array
        int[] array1 = {1, 2, 3};
        int[] array2 = {1, 2, 3};
        int[] array3 = {1, 2, 4};
        
        System.out.println("array1 sama dengan array2: " + Arrays.equals(array1, array2));  // true
        System.out.println("array1 sama dengan array3: " + Arrays.equals(array1, array3));  // false
        
        // Mengisi array dengan nilai tertentu
        int[] filled = new int[5];
        Arrays.fill(filled, 10);
        System.out.println("Array setelah diisi: " + Arrays.toString(filled));
        
        // Menyalin array
        int[] source = {1, 2, 3, 4, 5};
        int[] dest = Arrays.copyOf(source, 7);  // Menyalin dan memperbesar ukuran
        System.out.println("Array hasil copy: " + Arrays.toString(dest));
        
        // Menyalin sebagian array
        int[] partial = Arrays.copyOfRange(source, 1, 4);  // Menyalin dari indeks 1 (inklusif) hingga 4 (eksklusif)
        System.out.println("Array hasil copy sebagian: " + Arrays.toString(partial));
    }
}
Catatan Penting
  • Array di Java memiliki ukuran tetap yang tidak dapat diubah setelah dibuat
  • Indeks array dimulai dari 0
  • Mengakses indeks di luar batas array akan menyebabkan ArrayIndexOutOfBoundsException
  • Properti length memberikan jumlah elemen dalam array
  • Jika Anda memerlukan array dengan ukuran dinamis, gunakan ArrayList atau collection lainnya

13. Percabangan dalam Java

Percabangan (branching) digunakan untuk mengeksekusi blok kode tertentu berdasarkan kondisi. Java menyediakan beberapa jenis percabangan untuk mengontrol alur program.

If-Else: Mengeksekusi blok kode berdasarkan kondisi

Statement if-else digunakan untuk mengeksekusi blok kode jika kondisi tertentu terpenuhi (true), dan blok kode lain jika kondisi tidak terpenuhi (false).

IfElseExample.java 
// If sederhana
int age = 18;
if (age >= 18) {
    System.out.println("Anda sudah dewasa");
}

// If-else
int score = 75;
if (score >= 60) {
    System.out.println("Anda lulus");
} else {
    System.out.println("Anda tidak lulus");
}

// If-else-if ladder
int nilai = 85;
if (nilai >= 90) {
    System.out.println("Nilai A");
} else if (nilai >= 80) {
    System.out.println("Nilai B");
} else if (nilai >= 70) {
    System.out.println("Nilai C");
} else if (nilai >= 60) {
    System.out.println("Nilai D");
} else {
    System.out.println("Nilai E");
}

// Nested if (if bersarang)
boolean isRegistered = true;
boolean hasCompletedPayment = true;

if (isRegistered) {
    if (hasCompletedPayment) {
        System.out.println("Anda dapat mengakses kursus");
    } else {
        System.out.println("Silakan selesaikan pembayaran");
    }
} else {
    System.out.println("Silakan registrasi terlebih dahulu");
}

Switch-Case: Alternatif untuk banyak kondisi

Statement switch-case digunakan sebagai alternatif untuk if-else-if ladder ketika kita perlu memeriksa nilai variabel terhadap beberapa nilai konstan.

SwitchCaseExample.java 
// Switch-case dengan int
int day = 3;
switch (day) {
    case 1:
        System.out.println("Senin");
        break;
    case 2:
        System.out.println("Selasa");
        break;
    case 3:
        System.out.println("Rabu");
        break;
    case 4:
        System.out.println("Kamis");
        break;
    case 5:
        System.out.println("Jumat");
        break;
    case 6:
        System.out.println("Sabtu");
        break;
    case 7:
        System.out.println("Minggu");
        break;
    default:
        System.out.println("Hari tidak valid");
}

// Switch-case dengan char
char grade = 'B';
switch (grade) {
    case 'A':
        System.out.println("Excellent!");
        break;
    case 'B':
    case 'C':
        System.out.println("Well done");
        break;
    case 'D':
        System.out.println("You passed");
        break;
    case 'F':
        System.out.println("Better try again");
        break;
    default:
        System.out.println("Invalid grade");
}

// Switch-case dengan String (Java 7+)
String role = "admin";
switch (role) {
    case "admin":
        System.out.println("Akses penuh");
        break;
    case "editor":
        System.out.println("Akses edit");
        break;
    case "user":
        System.out.println("Akses terbatas");
        break;
    default:
        System.out.println("Akses ditolak");
}

Ternary Operator: Shorthand untuk if-else

Operator ternary (? :) adalah shorthand untuk statement if-else sederhana. Format: condition ? expression1 : expression2

TernaryOperatorExample.java 
// If-else biasa
int age = 17;
String status;
if (age >= 18) {
    status = "Dewasa";
} else {
    status = "Anak-anak";
}
System.out.println(status);

// Menggunakan ternary operator
int score = 75;
String result = (score >= 60) ? "Lulus" : "Tidak lulus";
System.out.println(result);

// Ternary operator bersarang
int nilai = 85;
String grade = (nilai >= 90) ? "A" : 
               (nilai >= 80) ? "B" : 
               (nilai >= 70) ? "C" : 
               (nilai >= 60) ? "D" : "E";
System.out.println(grade);
Tips Memilih Jenis Percabangan
  • Gunakan if untuk kondisi sederhana
  • Gunakan if-else untuk dua kemungkinan kondisi
  • Gunakan if-else-if untuk multiple kondisi dengan rentang nilai atau kondisi kompleks
  • Gunakan switch-case untuk multiple kondisi dengan nilai konstan
  • Gunakan ternary operator untuk if-else sederhana yang menghasilkan nilai

14. Array vs ArrayList

Array dan ArrayList adalah dua struktur data yang digunakan untuk menyimpan kumpulan elemen, tetapi keduanya memiliki perbedaan signifikan dalam hal fleksibilitas, fungsionalitas, dan penggunaan.

Array: Ukuran tetap, lebih efisien

  • Ukuran Tetap: Ukuran array ditentukan saat pembuatan dan tidak dapat diubah
  • Efisiensi: Lebih efisien dalam hal penggunaan memori dan kecepatan akses
  • Tipe Data: Dapat menyimpan tipe data primitif dan objek
  • Sintaks: Sintaks lebih sederhana dan langsung
  • Fungsionalitas: Tidak memiliki method bawaan untuk manipulasi data

ArrayList: Dinamis, mudah dimanipulasi

  • Ukuran Dinamis: Ukuran ArrayList dapat bertambah atau berkurang secara dinamis
  • Fleksibilitas: Lebih fleksibel dengan banyak method untuk manipulasi data
  • Tipe Data: Hanya dapat menyimpan objek (tidak dapat menyimpan tipe data primitif secara langsung)
  • Sintaks: Memerlukan import dari java.util.ArrayList
  • Fungsionalitas: Memiliki banyak method bawaan seperti add(), remove(), contains(), dll.
ArrayVsArrayList.java 
import java.util.ArrayList;
import java.util.Arrays;

public class ArrayVsArrayList {
    public static void main(String[] args) {
        // Array
        System.out.println("=== Array ===");
        
        // Inisialisasi array
        int[] numbersArray = new int[5];
        
        // Mengisi array
        for (int i = 0; i < numbersArray.length; i++) {
            numbersArray[i] = i * 10;
        }
        
        // Menampilkan array
        System.out.println("Array: " + Arrays.toString(numbersArray));
        
        // Mengakses elemen array
        System.out.println("Elemen ke-2: " + numbersArray[2]);
        
        // Mengubah elemen array
        numbersArray[3] = 35;
        System.out.println("Array setelah diubah: " + Arrays.toString(numbersArray));
        
        // Tidak bisa menambah elemen baru ke array yang sudah ada
        // Harus membuat array baru dengan ukuran yang lebih besar
        int[] newArray = new int[numbersArray.length + 1];
        System.arraycopy(numbersArray, 0, newArray, 0, numbersArray.length);
        newArray[newArray.length - 1] = 50;
        System.out.println("Array baru: " + Arrays.toString(newArray));
        
        // ArrayList
        System.out.println("\n=== ArrayList ===");
        
        // Inisialisasi ArrayList
        ArrayList numbersList = new ArrayList<>();
        
        // Menambahkan elemen ke ArrayList
        for (int i = 0; i < 5; i++) {
            numbersList.add(i * 10);
        }
        
        // Menampilkan ArrayList
        System.out.println("ArrayList: " + numbersList);
        
        // Mengakses elemen ArrayList
        System.out.println("Elemen ke-2: " + numbersList.get(2));
        
        // Mengubah elemen ArrayList
        numbersList.set(3, 35);
        System.out.println("ArrayList setelah diubah: " + numbersList);
        
        // Menambahkan elemen baru ke ArrayList
        numbersList.add(50);
        System.out.println("ArrayList setelah ditambah: " + numbersList);
        
        // Menghapus elemen dari ArrayList
        numbersList.remove(1);  // Menghapus elemen di indeks 1
        System.out.println("ArrayList setelah dihapus: " + numbersList);
        
        // Memeriksa apakah elemen ada di ArrayList
        boolean contains = numbersList.contains(30);
        System.out.println("ArrayList berisi 30? " + contains);
        
        // Mendapatkan ukuran ArrayList
        System.out.println("Ukuran ArrayList: " + numbersList.size());
        
        // Mengosongkan ArrayList
        numbersList.clear();
        System.out.println("ArrayList setelah dikosongkan: " + numbersList);
    }
}

Kapan Menggunakan Array vs ArrayList

Pilihan antara Array dan ArrayList tergantung pada kebutuhan spesifik aplikasi Anda:

Gunakan Array ketika Gunakan ArrayList ketika
Ukuran koleksi tetap dan diketahui sebelumnya Ukuran koleksi dapat berubah selama runtime
Performa adalah prioritas utama Fleksibilitas lebih penting daripada performa
Menyimpan tipe data primitif (tanpa autoboxing) Memerlukan banyak operasi manipulasi data (add, remove, dll.)
Memerlukan array multi-dimensi Memerlukan fitur-fitur collection seperti sorting, searching, dll.
Catatan Penting

ArrayList menggunakan array di balik layar, tetapi secara otomatis menangani resizing saat diperlukan. Ini memberikan fleksibilitas dengan sedikit overhead performa. Jika Anda tidak yakin mana yang harus digunakan, ArrayList biasanya merupakan pilihan yang lebih aman karena fleksibilitasnya.

15. Konsep OOP dalam Java

Java adalah bahasa pemrograman berorientasi objek (Object-Oriented Programming/OOP). OOP adalah paradigma pemrograman yang menggunakan "objek" untuk mendesain aplikasi dan program. Ada empat konsep utama dalam OOP: Enkapsulasi, Inheritance, Polymorphism, dan Abstraksi.

Enkapsulasi: Melindungi data dengan modifier akses

Enkapsulasi adalah konsep menyembunyikan detail implementasi dan hanya mengekspos fungsionalitas yang diperlukan. Ini dicapai dengan menggunakan modifier akses dan method getter/setter.

Encapsulation.java 
public class Person {
    // Private fields (data tersembunyi)
    private String name;
    private int age;
    
    // Constructor
    public Person(String name, int age) {
        this.name = name;
        this.setAge(age);  // Menggunakan setter untuk validasi
    }
    
    // Getter methods
    public String getName() {
        return name;
    }
    
    public int getAge() {
        return age;
    }
    
    // Setter methods dengan validasi
    public void setName(String name) {
        if (name != null && !name.isEmpty()) {
            this.name = name;
        }
    }
    
    public void setAge(int age) {
        if (age >= 0 && age <= 120) {  // Validasi umur
            this.age = age;
        } else {
            throw new IllegalArgumentException("Umur harus antara 0 dan 120");
        }
    }
    
    public static void main(String[] args) {
        Person person = new Person("John", 25);
        
        // Mengakses data melalui getter
        System.out.println("Nama: " + person.getName());
        System.out.println("Umur: " + person.getAge());
        
        // Mengubah data melalui setter
        person.setName("John Doe");
        person.setAge(30);
        
        System.out.println("Nama baru: " + person.getName());
        System.out.println("Umur baru: " + person.getAge());
        
        // Ini akan menyebabkan error karena validasi
        // person.setAge(150);
    }
}

Inheritance: Pewarisan sifat dari kelas induk ke kelas anak

Inheritance (pewarisan) memungkinkan sebuah class untuk mewarisi properti dan method dari class lain. Ini memungkinkan reusability kode dan hubungan hierarki antar class.

Inheritance.java 
// Class induk (parent/super class)
class Animal {
    protected String name;
    protected int age;
    
    public Animal(String name, int age) {
        this.name = name;
        this.age = age;
    }
    
    public void eat() {
        System.out.println(name + " sedang makan");
    }
    
    public void sleep() {
        System.out.println(name + " sedang tidur");
    }
}

// Class anak (child/sub class) yang mewarisi Animal
class Dog extends Animal {
    private String breed;
    
    public Dog(String name, int age, String breed) {
        super(name, age);  // Memanggil constructor class induk
        this.breed = breed;
    }
    
    // Method tambahan khusus untuk Dog
    public void bark() {
        System.out.println(name + " menggonggong: Woof! Woof!");
    }
    
    // Override method dari class induk
    @Override
    public void eat() {
        System.out.println(name + " sedang makan tulang");
    }
}

// Class anak lain yang mewarisi Animal
class Bird extends Animal {
    private boolean canFly;
    
    public Bird(String name, int age, boolean canFly) {
        super(name, age);
        this.canFly = canFly;
    }
    
    // Method tambahan khusus untuk Bird
    public void fly() {
        if (canFly) {
            System.out.println(name + " sedang terbang");
        } else {
            System.out.println(name + " tidak bisa terbang");
        }
    }
    
    // Override method dari class induk
    @Override
    public void eat() {
        System.out.println(name + " sedang makan biji-bijian");
    }
}

public class InheritanceDemo {
    public static void main(String[] args) {
        // Membuat objek dari class induk
        Animal animal = new Animal("Hewan", 5);
        animal.eat();  // Output: Hewan sedang makan
        
        // Membuat objek dari class anak
        Dog dog = new Dog("Buddy", 3, "Golden Retriever");
        dog.eat();     // Output: Buddy sedang makan tulang (method yang di-override)
        dog.sleep();   // Output: Buddy sedang tidur (method dari class induk)
        dog.bark();    // Output: Buddy menggonggong: Woof! Woof! (method khusus Dog)
        
        Bird bird = new Bird("Tweety", 2, true);
        bird.eat();    // Output: Tweety sedang makan biji-bijian (method yang di-override)
        bird.sleep();  // Output: Tweety sedang tidur (method dari class induk)
        bird.fly();    // Output: Tweety sedang terbang (method khusus Bird)
    }
}

Polymorphism: Kemampuan metode dengan nama sama berperilaku berbeda

Polymorphism (polimorfisme) memungkinkan objek dari class yang berbeda untuk merespons method yang sama dengan cara yang berbeda. Ada dua jenis polimorfisme: compile-time (overloading) dan runtime (overriding).

Polymorphism.java 
// Compile-time Polymorphism (Method Overloading)
class Calculator {
    // Method dengan nama sama tetapi parameter berbeda
    public int add(int a, int b) {
        return a + b;
    }
    
    public double add(double a, double b) {
        return a + b;
    }
    
    public int add(int a, int b, int c) {
        return a + b + c;
    }
}

// Runtime Polymorphism (Method Overriding)
class Shape {
    public double calculateArea() {
        return 0;  // Default implementation
    }
    
    public void display() {
        System.out.println("Ini adalah bentuk");
    }
}

class Circle extends Shape {
    private double radius;
    
    public Circle(double radius) {
        this.radius = radius;
    }
    
    @Override
    public double calculateArea() {
        return Math.PI * radius * radius;
    }
    
    @Override
    public void display() {
        System.out.println("Ini adalah lingkaran dengan radius " + radius);
    }
}

class Rectangle extends Shape {
    private double width;
    private double height;
    
    public Rectangle(double width, double height) {
        this.width = width;
        this.height = height;
    }
    
    @Override
    public double calculateArea() {
        return width * height;
    }
    
    @Override
    public void display() {
        System.out.println("Ini adalah persegi panjang dengan lebar " + width + " dan tinggi " + height);
    }
}

public class PolymorphismDemo {
    public static void main(String[] args) {
        // Compile-time Polymorphism (Method Overloading)
        Calculator calc = new Calculator();
        System.out.println("1 + 2 = " + calc.add(1, 2));
        System.out.println("1.5 + 2.5 = " + calc.add(1.5, 2.5));
        System.out.println("1 + 2 + 3 = " + calc.add(1, 2, 3));
        
        // Runtime Polymorphism (Method Overriding)
        Shape shape1 = new Circle(5);
        Shape shape2 = new Rectangle(4, 6);
        
        // Memanggil method yang sama tetapi implementasi berbeda
        shape1.display();  // Output: Ini adalah lingkaran dengan radius 5
        shape2.display();  // Output: Ini adalah persegi panjang dengan lebar 4 dan tinggi 6
        
        System.out.println("Luas lingkaran: " + shape1.calculateArea());
        System.out.println("Luas persegi panjang: " + shape2.calculateArea());
        
        // Polymorphism dengan array
        Shape[] shapes = new Shape[3];
        shapes[0] = new Circle(3);
        shapes[1] = new Rectangle(5, 4);
        shapes[2] = new Circle(7);
        
        for (Shape shape : shapes) {
            shape.display();
            System.out.println("Luas: " + shape.calculateArea());
            System.out.println();
        }
    }
}

Abstraksi: Menyembunyikan detail implementasi

Abstraksi adalah konsep menyembunyikan detail implementasi dan hanya menunjukkan fungsionalitas yang penting kepada pengguna. Ini dapat dicapai melalui abstract class dan interface.

Abstraction.java 
// Abstract Class
abstract class Vehicle {
    protected String brand;
    protected String model;
    
    public Vehicle(String brand, String model) {
        this.brand = brand;
        this.model = model;
    }
    
    // Abstract method (harus diimplementasikan oleh subclass)
    public abstract void start();
    
    // Concrete method (implementasi umum)
    public void stop() {
        System.out.println("Kendaraan berhenti");
    }
    
    public void displayInfo() {
        System.out.println("Brand: " + brand);
        System.out.println("Model: " + model);
    }
}

class Car extends Vehicle {
    private int numDoors;
    
    public Car(String brand, String model, int numDoors) {
        super(brand, model);
        this.numDoors = numDoors;
    }
    
    @Override
    public void start() {
        System.out.println("Mobil dinyalakan dengan kunci");
    }
    
    public void honk() {
        System.out.println("Beep! Beep!");
    }
}

class Motorcycle extends Vehicle {
    private boolean hasSidecar;
    
    public Motorcycle(String brand, String model, boolean hasSidecar) {
        super(brand, model);
        this.hasSidecar = hasSidecar;
    }
    
    @Override
    public void start() {
        System.out.println("Motor dinyalakan dengan kick starter");
    }
    
    public void wheelie() {
        System.out.println("Motor melakukan wheelie!");
    }
}

// Interface
interface Flyable {
    void fly();  // Abstract method (implisit)
    void land();
    
    // Default method (Java 8+)
    default void takeOff() {
        System.out.println("Lepas landas...");
    }
}

interface Swimmable {
    void swim();
}

// Class yang mengimplementasikan interface
class Bird implements Flyable {
    private String species;
    
    public Bird(String species) {
        this.species = species;
    }
    
    @Override
    public void fly() {
        System.out.println(species + " terbang dengan sayap");
    }
    
    @Override
    public void land() {
        System.out.println(species + " mendarat di pohon");
    }
}

// Class yang mengimplementasikan multiple interfaces
class Duck implements Flyable, Swimmable {
    @Override
    public void fly() {
        System.out.println("Bebek terbang rendah di atas air");
    }
    
    @Override
    public void land() {
        System.out.println("Bebek mendarat di air");
    }
    
    @Override
    public void swim() {
        System.out.println("Bebek berenang di danau");
    }
}

public class AbstractionDemo {
    public static void main(String[] args) {
        // Abstract Class
        // Vehicle vehicle = new Vehicle("Generic", "Model");  // Error: Cannot instantiate abstract class
        
        Car car = new Car("Toyota", "Corolla", 4);
        car.displayInfo();
        car.start();  // Output: Mobil dinyalakan dengan kunci
        car.stop();   // Output: Kendaraan berhenti
        car.honk();   // Output: Beep! Beep!
        
        Motorcycle motorcycle = new Motorcycle("Honda", "CBR", false);
        motorcycle.displayInfo();
        motorcycle.start();  // Output: Motor dinyalakan dengan kick starter
        motorcycle.stop();   // Output: Kendaraan berhenti
        motorcycle.wheelie();  // Output: Motor melakukan wheelie!
        
        // Interface
        Flyable eagle = new Bird("Eagle");
        eagle.takeOff();  // Output: Lepas landas...
        eagle.fly();      // Output: Eagle terbang dengan sayap
        eagle.land();     // Output: Eagle mendarat di pohon
        
        Duck duck = new Duck();
        duck.takeOff();  // Output: Lepas landas...
        duck.fly();      // Output: Bebek terbang rendah di atas air
        duck.land();     // Output: Bebek mendarat di air
        duck.swim();     // Output: Bebek berenang di danau
    }
}
Perbedaan Abstract Class dan Interface
Abstract Class Interface
Dapat memiliki method abstract dan non-abstract Semua method abstract (kecuali default dan static method di Java 8+)
Dapat memiliki variabel instance Hanya dapat memiliki konstanta (public static final)
Dapat memiliki constructor Tidak dapat memiliki constructor
Subclass hanya dapat extends satu abstract class Class dapat implements multiple interfaces
Digunakan ketika ada hubungan "is-a" Digunakan ketika ada hubungan "can-do"
